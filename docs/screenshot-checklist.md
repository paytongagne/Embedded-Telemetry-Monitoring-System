# Screenshot Checklist

Use this checklist when preparing project screenshots for the README or portfolio site.

## Recommended screenshots

1. FastAPI docs page at `/docs`
2. `GET /api/v1/summary` response
3. `GET /api/v1/devices` response
4. `GET /api/v1/alerts` response
5. Dashboard overview with summary cards
6. Terminal output showing tests passing
7. Project folder structure in editor

## Suggested captions

- API documentation generated by FastAPI
- Fleet summary response after seeding demo data
- Device status table with normal, warning, and critical states
- Active alert records generated by telemetry classification
- Dashboard view connected to the local API
- Automated test run for model, classifier, storage, and API behavior

## Notes

Keep screenshots focused on the project. Avoid including personal browser tabs, tokens, local paths with sensitive names, or unrelated applications.
