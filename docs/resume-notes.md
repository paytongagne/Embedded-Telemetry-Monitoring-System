# Resume Notes

## Strong Project Summary

Built an embedded telemetry monitoring backend that simulates connected-device readings, validates telemetry payloads, classifies device health, stores readings in SQLite, and exposes data through FastAPI endpoints.

## Resume Bullets

- Built a Python telemetry monitoring system using FastAPI, Pydantic, and SQLite to simulate connected-device health tracking.
- Implemented health classification logic for temperature, voltage, battery level, signal strength, and memory usage.
- Designed a relational database schema for devices, telemetry readings, and alerts with query support for latest readings and device summaries.
- Added a dashboard concept to visualize fleet health, recent readings, and alert-driven system status.
- Documented architecture, API usage, database schema, and roadmap for a workplace-ready GitHub portfolio project.

## Interview Story

I built this project to show how device-style data can move through a backend system. The application generates simulated telemetry, validates the payload, classifies device health, stores readings in a database, and exposes the results through API endpoints. I structured the repo like a real engineering project with models, services, storage, tests, scripts, and documentation.
